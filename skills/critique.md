# Skill: critique

Activate when the designer says "run critique".
Ask three things before starting:

1. "Is there a design brief or problem statement I should reference?
   Paste it, share a file path, or describe the core problem —
   this helps me critique against intent, not just aesthetics."

2. "Who do you want in the room? Available personas:
   - **Alex** — Peer Designer (craft, pattern consistency, edge cases, state completeness)
   - **Jordan** — PM Stakeholder (outcomes, scope, engineering buy-in, timeline)
   - **Sam** — Engineering Lead (feasibility, reuse, maintainability, spec clarity)

   If you've added personas from resources/personas/ — including any
   generated with persona-gen — name them and I'll bring them in too.

   Options:
   - Full room (Alex, Jordan, Sam)
   - Specific members — name them
   - Single voice for a deep take"

3. "What are you sharing — prototype link, flow
   description, or screens?"
